Domino / Notes Web AP 現代化

讓 IE 時代的 Notes 表單，在 Edge / Chrome 重新可用

不是一個個盲改。先系統化盤點、再實機驗證，最後用一支共用相容墊片把絕大多數問題集中處理；剩下少數才需補環境或小幅改寫。可控、可重複、可交付。

一、方法論：四階段

準備

匯出與版控

把系統匯出成可分析的設計檔，納入版控，建立可回復基準。

靜態盤點

找出 IE 寫法

掃描全部設計元素，依風險與修正難度分類產出報告。只讀不改。

瀏覽器驗證

確認真的壞

實機操作，分清真損壞與誤判，留下截圖與錯誤證據。

修復

共用墊片

一支相容墊片，呼叫端零改動；分批修、人工複查、測試環境驗證。

核心病灶：「選人彈窗」依賴兩個 IE 專屬技術。先用 showModalDialog 開通訊錄，彈窗內再用 ActiveX 同步 AJAX 向後端取人員。整條資料鏈在現代瀏覽器全部失效。

三、逐層修復，以「選人功能」為例

一支共用墊片、呼叫端不改。每修好一層，就精準暴露下一個真問題，系統化地一層層解開。

起點：注入共用相容墊片共用墊片

把 IE 的 ActiveX / showModalDialog 用標準技術補回。選人彈窗開啟、現有人員載入。

第 1 層單位下拉空白，顯示「File does not exist」環境／資料

下拉用公式讀外部人事庫，但測試環境尚未佈署該庫。補上人事資料後正常顯示。

第 2 層請求被瀏覽器封鎖（Mixed Content）共用墊片

舊程式寫死 http 呼叫後端，HTTPS 頁面被封鎖。墊片自動把同站 http 升級 https。

第 3 層解析回傳資料失敗（selectNodes 不存在）共用墊片

程式用 IE 專屬 selectNodes 解析 XML。墊片以標準方法補回，待選清單載出。

第 4 層按「確定」未把值帶回需重新設計

呼叫端依賴 showModalDialog 的同步阻塞語意，現代瀏覽器無法模擬。需小幅改寫為非同步回呼。唯一需動邏輯的一類。

四、所有問題，歸三類處理

多數

機械式 IE 語法

一支共用墊片集中修，呼叫端零改動
  • ActiveX 同步 AJAX
  • selectNodes / node.text
  • http 升級 https
  • showModalDialog 開窗與回填
佈署

環境／資料依賴

不是程式問題，由環境補齊
  • 外部人事庫未佈署
  • 簽核／流程模組未就緒
  • 跨庫設定文件缺漏
少數

需重新設計

需小幅改寫程式邏輯
  • showModalDialog 同步回呼
  • 改為非同步 callback
  • 一次改好，同模式可複用

五、完整 = 功能正常 + 介面到位

功能修好只是「能用」。要「完整」，還要把舊時代的版面一併現代化。以選人視窗為例：
